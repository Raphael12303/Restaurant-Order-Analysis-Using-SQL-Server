This project explores and analyzes a restaurant database using Microsoft SQL Server and SQL Server Management Studio. The goal of the project is to understand menu structure, pricing patterns, and customer ordering behavior by writing and executing SQL queries on relational tables.

The dataset contains two main tables: menu_items, which includes information about dishes, categories, and prices, and order_details, which stores individual order transactions. Using SQL, the project performs data exploration and analysis to generate useful business insights from the restaurant’s operations.

Several analytical queries were written to investigate the dataset. The analysis begins with exploring the menu_items table to determine the total number of dishes available, identify the least and most expensive items, and analyze how dishes are distributed across different categories. Additional queries calculate the number of dishes in each category and determine the average price per category to understand pricing trends.

The project also focuses on order analysis using the order_details table. Queries were used to determine the total number of orders placed, the total number of items ordered, and the overall date range of the dataset. Further analysis identifies which orders contained the highest number of items and how many orders included more than twelve items.

To gain deeper insights, a JOIN operation was performed between the menu_items and order_details tables. This allowed the analysis of item popularity, calculation of total order values, and identification of the highest-value order placed in the restaurant.

Overall, this project demonstrates practical SQL skills such as data exploration, table joins, aggregation, grouping, filtering, and sorting. It highlights how SQL can be used to analyze operational data and generate insights that can support decision-making in a restaurant business environment.
